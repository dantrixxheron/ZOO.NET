# ZOO.NET
Zoo.net: El primer Zoológico digital.
Página acerca de animales desarrollado en el Summer Camp: Tecnolochicas pro 2023.

## Intro 📚
Este proyecto consistió en una práctica realizada en el bootcamp de Tecnolochicas Pro, donde se nos enseñaba acerca de los componentes básicos de una página web con HTML y CSS. A partir de esta información creé y diseñé Zoo.net lo que se supone que es un zoológico digital.

### ¿Por qué utilizar HTML y CSS en vez de otra cosa?
Como parte del curso de inicialización a la programación web, fue fundamental conocer la sintaxis básica para desarrollar páginas web funcionales y darle estilos a estas mismas, por ende, HTML es un recurso fundamental para el desarrollo de páginas web y junto con CSS se le pueden dar diseños personalizados y de alta calidad, a tal nivel de poder replicar de manera exacta la página ya conocida que es Google.com al menos en interfáz.

## ¿Qué construí? ❇️
Se construyó una interfaz acorde a un sitio web acerca de animales la cual estuvo divida en 4 páginas de HTML principales: 

### 1. Página index.html:
- Diseño de la barra de navegación utilizada en todas las páginas.
- Información acerca de Zoo.net y dar la bienvenida a los usuarios.
- Logo de la plataforma desarrollado por Playground AI.
- Algunos de los animales que vienen en el la plataforma (la idea es que posteriormente estén en el index los animales más buscados).
- Botón para continuar ("Ver más") que direcciona a la página siguiente.

### 2. Página animales.html:
Se accede a esta página por medio del enlace en la barra de navegación o por medio del botón "Ver más" en el index.html.
- Contiene la barra de navegación ya diseñada.
- Se inicia un contenedor que tiene a los animales.
- Un título para la sección iniciada (ya sea animales de x especie o así).
- Un grid de 3 elementos con similar estructura que en index.html
*Se repiten las veces necesarias los dos últimos elementos.

### 3. Página animalN.html:
Titulé esta sección así ya que todos los animales existentes tienen el nombre de dicho animal pero siguien la misma estructura (estructura que también se puede ver en structure.html en caso de querer agregar otro animal) pero se entiende por animalN.html a todos los animales compuestos en la página (leon.html, cebra.html, jirafa.html, etc.):
- Barra de navegación.
- Título donde se pone el nombre del animal en cuestión.
- un grid de dos elementos que contiene: descripción de la página a mostrar y una imagen del animal.
- Título de la siguiente sección "datos curiosos sobre ..."
- Un grid de 3 elementos por fila que contienen la información correspondiente seguido de una imagen ilustrativa. (Se repite 2 veces para que san 6 y en la última se tiene un enlace para conocer más del animal (enlace que se direccióna en código js llamado directions.js))
- Título centrado de la siguiente sección "Video informativo".
- Fuente de un video de youtube con información especie documental acerca del animal en cuestión.
- Título de la siguiente sección "Referencias"
- Referencias en formato lista (y versión APA 7).

### 4. Página about.html
A esta página se puede acceder únicamente mediante la barra de navegación en el enlace de "Acerca de".
- Barra de navegación.
- Título de la página.
- Un grid de 2 elementos que contiene la descripción de esta página (dar una descripción del proyecto) y la imagen del logo de la página desarrollado por Playground AI.
- Título de la siguiente sección "Propósito de Zoo.net"
- Un grid de 3 elementos por fila (y dos filas) que contienen los objetivos del desarrollo de este proyecto seguido del enlace a este repositorio para añadir elementos y de un enlace para donar para salvaguardar las zonas verdes en el mundo.
- Un footer que contiene las diversas formas para contactar al desarrollador del proyecto.
